## 一、为什么选择 EasyOCR 而不是 Tesseract

| 对比维度 | Tesseract 训练 | EasyOCR 微调 |
|----------|---------------|-------------|
| 框架 | C++ 自研引擎 | Python + PyTorch |
| 训练流程 | `.tif → .box → .lstmf → tesstrain` 多步转换 | PyTorch 端到端训练，CTC Loss + Adam |
| ML 学习价值 | 低（黑盒流水线，调参空间小） | 高（可见 CRNN 结构、梯度反向传播、迁移学习） |
| 推理速度 | 快（C++） | 中等（Python） |
| 你的场景适配 | 需要从头搭建训练管线 | 预训练模型直接可用，微调即可 |

**结论：EasyOCR 微调。** 它的预训练模型是基于 MJSynth（900万张合成图）从零训练的，和我们做的事情一样——只不过我们用 Exocet 字体来微调，而不需要几百万张图和几周 GPU 时间。

## 二、EasyOCR 的预训练模型是怎么来的（和 Tesseract 无关）

```
文本语料库 (维基/词典)  +  字体库 (几十种系统字体)  +  背景图库 (自然场景)
                │                      │                      │
                └──────────────────────┼──────────────────────┘
                                       ▼
                          ┌────────────────────────┐
                          │  渲染引擎 → 合成标注数据  │
                          │  "Hello" → [图片, label] │
                          │  生成 900万+ 张图         │
                          └───────────┬────────────┘
                                      ▼
                          ┌────────────────────────┐
                          │  CRNN 模型 + CTC Loss    │
                          │  多 GPU 训练数天到数周    │
                          └───────────┬────────────┘
                                      ▼
                          ┌────────────────────────┐
                          │  发布 .pth 权重          │
                          │  (pip install 拿到)      │
                          └────────────────────────┘
```

**EasyOCR 和 Tesseract 没有任何关系。** 它们是两条完全独立的路线：Tesseract 是 Google 的 C++ OCR 引擎，EasyOCR 是 JaidedAI 开源的 PyTorch OCR 工具包。

## 三、从零训练 vs 微调：多费劲？

| | 从零训练 | 微调（我们的路线） |
|---|---------|-----------------|
| 需要数据量 | 几百万张图 | 几千~几万张合成图 |
| GPU 时间 | 数天到数周 | 几十分钟到几小时 |
| 超参调优 | 大量试验 | 默认值即可 |
| 最终效果 | 大概率不如预训练模型 | 在特定字体上 > 预训练 |

**比喻：** 从零训练 = 自己种麦子 → 磨面粉 → 烤面包；微调 = 买现成面包片，自己加料烤。

CNN 前几层学的是**通用视觉底层特征**（边缘、拐角、纹理），对所有字体都一样。微调时冻结这些层，只训练后面的 RNN/Head 来适应 Exocet 的特殊字形，这正是迁移学习的核心思想。

## 四、CRNN 模型三层结构详解

EasyOCR 的识别器（Recognizer）是一个 **CRNN** 模型：

```
输入图片 (32×128 像素)
        │
        ▼
┌───────────────────────────────────────────┐
│ ① CNN 卷积层 (Backbone / 脊椎)              │
│                                            │
│ 做什么：从像素中提取视觉特征                    │
│ 学什么：「哪里有竖线？哪里有圆角？这里有拐弯吗？」    │
│                                            │
│ 结构：Conv2d → ReLU → MaxPool → Conv2d → ... │
│ 输出：特征图 → 压平为序列 (每列 = 一个特征向量)    │
│                                            │
│ 关键概念：                                   │
│   · 卷积核 (Kernel)：一个小窗口在图片上滑动      │
│   · 感受野 (Receptive Field)：一个神经元能看到的  │
│     原始图片范围，越深越大                      │
│   · 池化 (Pooling)：缩小尺寸，保留重要特征       │
│                                            │
│ 类比：小孩学看东西——横竖撇捺都是笔画              │
└──────────────────┬────────────────────────┘
                   │ 特征序列 (B, 512, 1, 32)
                   ▼
┌───────────────────────────────────────────┐
│ ② RNN 循环层 (Neck / 脖子)                   │
│                                            │
│ 做什么：理解时序上的上下文关系                  │
│ 学什么：「前面的竖线+上面有个点+很窄 → 可能是 i」  │
│                                            │
│ 结构：双向 GRU (Gated Recurrent Unit)        │
│                                            │
│ 关键概念：                                   │
│   · 隐藏状态 (Hidden State)：RNN 的记忆，      │
│     把前面看到的信息编码成一个向量传给下一步      │
│   · GRU 门控：更新门(记住什么) + 重置门(忘掉什么) │
│   · 双向 (Bidirectional)：从左往右 + 从右往左   │
│     各跑一遍，合并信息 → 当前字符既看到前面也看到  │
│     后面是什么，判断更准                       │
│   · 和 LSTM 的区别：GRU 少一个门，参数更少      │
│     训练更快，效果相当                         │
│                                            │
│ 类比：小孩学认字——知道笔画了，把笔画组合成字      │
└──────────────────┬────────────────────────┘
                   │ 序列特征 (B, 32, 256)
                   ▼
┌───────────────────────────────────────────┐
│ ③ Head 分类头 (Head / 头部)                  │
│                                            │
│ 做什么：把特征向量映射到具体字符               │
│ 学什么：「这个 256 维向量 = 字符 a」            │
│                                            │
│ 结构：nn.Linear(256, 97)  ← 一个矩阵乘法      │
│                                            │
│ 本质：W(97×256) × feature(256) + b(97)       │
│       = 97 个字符的得分/概率                  │
│                                            │
│ 97 = 26小写 + 26大写 + 10数字 + 符号 + 1 blank  │
│ 我们可以改成 43 (= 42字符 + 1 blank)           │
│                                            │
│ 类比：小孩认出这字了，念出这是什么字             │
└───────────────────────────────────────────┘
```

**Head 就是一个矩阵乘法，参数量很小：**
- 原版：256 × 97 + 97 ≈ **2.5 万参数**
- 缩减版：256 × 43 + 43 ≈ **1.1 万参数**

## 五、微调时的分层冻结策略

训练时通过 `param.requires_grad = False` 控制哪些层更新：

```
                         CNN (特征提取)    RNN (上下文)    Head (分类)
                         ────────────    ───────────    ──────────
方案A：数据少(<5000)        ❄️ 冻结         🔥 训练         🔥 训练
  → 最快收敛，不易过拟合

方案B：数据多(>50000)       🔥 训练         🔥 训练         🔥 训练
  → 最能适应新字体，但需要更多数据

方案C：折中（推荐）         前几层❄️ + 后几层🔥  🔥 训练     🔥 训练
  → 保留通用边缘检测，让高层适应 Exocet 的特殊字形
```

**我们先用方案 C。** 理由：Exocet 和标准英文差异大，但合成数据量不会太多，全解冻容易过拟合。

## 六、i / 1 / j 和 0 / o 为什么混淆？

根因不在 CNN，而在 **RNN + Head**：

- CNN 能正常提取特征（它知道这里有笔画）
- 但预训练模型的 RNN+Head 是用 Arial / Times New Roman 等标准字体训练的
- 它学到的映射是：「标准字体的特征 → 字符」
- Exocet 的 i 更窄、点和杆间距不同 → RNN+Head 把特征**映射错了字符**

**微调就是让 RNN+Head 重新学习：**
> 「Exocet 字体里这种特殊的时空特征 → 正确的字符」

此外**缩减字符集**也有帮助：
- 如果原来要在 97 个候选里做选择，容易把 i 当成 j
- 如果候选只有 42 个，决策空间小，分类更准

## 七、字符集缩减

EasyOCR 默认字符集 ~97 个，我们可以缩减到你的 42 个。Head 层自动适配：

```python
# 原来: 97 个字符
chars = "abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789..."

# 缩减: 只保留你场景中出现的字符
chars = "ARMO SWRD+:-!0123456789"  # ← 你从图片中收集
# Head 自动变为 nn.Linear(256, len(chars)+1)  ← +1 是 CTC blank
```

**好处：参数量减半、收敛更快、混淆更少。**

## 八、数据增强 vs 预处理

| | 数据增强（自然场景） | 预处理（我们的场景） |
|---|---|---|
| 什么 | 加噪声、旋转、模糊 | 二值化、灰度化、拉对比度 |
| 为什么 | 模拟现实光照/角度干扰 | 你的图是屏幕渲染的，本身干净 |
| 策略 | 先不做，有需要再加 | 先用，让输入更清晰 |

**我们的流程：**
```
原始 PNG 截图 → 灰度化 → 对比度拉伸/二值化 → 送入识别器
```

## 十一、核心 ML 概念速查

| 概念 | 一句话解释 |
|------|-----------|
| **迁移学习** | 用别人训练好的模型做基础，在自己的数据上微调 |
| **过拟合** | 模型背下了训练数据，但无法泛化到新图片 |
| **卷积核** | 一个小矩阵在图片上滑动，检测特定模式（边缘/纹理） |
| **感受野** | 一个神经元「看到」的原始图片范围 |
| **池化** | 缩小特征图尺寸，保留重要信息，减少计算量 |
| **隐藏状态** | RNN 的记忆，把历史信息编码成向量传到下一步 |
| **GRU 门控** | 更新门（记住新信息）+ 重置门（忘掉旧信息） |
| **CTC Loss** | 特殊损失函数，不需要每个像素对齐到字符，自动学习对齐 |
| **CTC blank** | 一个特殊标记，表示「这里没有字符」，夹在重复字符之间 |
| **反向传播** | 从损失值出发，用链式法则计算每个参数的梯度 |
| **梯度下降** | 沿着梯度负方向更新参数，让模型越来越准确 |
| **学习率** | 每次参数更新的步长，太大跳过去太小太慢 |
| **冻结层** | `requires_grad=False`，该层参数不参与更新 |
| **Head** | 网络的最后一层，负责做最终分类决策 |
| **Epoch** | 把全部训练数据过一遍 = 一个 epoch |
